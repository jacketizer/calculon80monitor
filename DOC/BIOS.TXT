Device Driver Header
--------------------

In the future it should be possible to load or configure custom device drivers
which would then require some kind of header in the driver image. The header
should atleast contain the following:

  * Device type
  * Number of instances/channels
  * Individual instance init/reconfig support?
  * Interrupt support?
  * Interrupt vectors
  * Length of driver image
  * Jump table to routines
